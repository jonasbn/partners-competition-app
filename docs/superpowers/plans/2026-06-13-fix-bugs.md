## Task 1: Fix total-games count in SimpleLeaderboard

**Files:**
- Modify: `src/components/SimpleLeaderboard.jsx:164`
- Modify: `src/test/components/Leaderboard.test.jsx`

### Background

`getLeaderboardData` (in `src/utils/dataUtils.js:92`) returns `{ players, games }`.
The component stores this in `leaderboardData` (line 16) and extracts only `players` (line 20).
The footer (line 164) currently computes total games as:

```js
Math.floor(players.reduce((sum, p) => sum + (p.gamesPlayed || 0), 0) / 2)
```

With 6 players each accumulating 1 `gamesPlayed` per game, this divides the true count by 3 (sums 6, divides by 2 = 3 instead of 1). The correct value is `leaderboardData.games.length`.

The existing test mock returns `{ players: [...] }` without a `games` array, which must be updated so the component doesn't crash on `.games.length`.

- [ ] **Step 1: Update the mock in Leaderboard.test.jsx to include a `games` array**

Open `src/test/components/Leaderboard.test.jsx`. The current mock (lines 8–16) is:

```js
vi.mock('../../utils/dataUtils', () => ({
  getLeaderboardData: () => ({
    players: [
      { id: 1, name: 'Jonas', cumulativeScore: 15, games: [{ gameId: 1, score: 3 }] },
      { id: 2, name: 'Torben', cumulativeScore: 12, games: [{ gameId: 1, score: 2 }] },
      { id: 3, name: 'Gitte', cumulativeScore: 10, games: [{ gameId: 1, score: 1 }] }
    ]
  })
}));
```

Replace it with:

```js
vi.mock('../../utils/dataUtils', () => ({
  getLeaderboardData: () => ({
    players: [
      { id: 1, name: 'Jonas', cumulativeScore: 15, gamesPlayed: 2, games: [{ gameId: 1, score: 3 }, { gameId: 2, score: 2 }] },
      { id: 2, name: 'Torben', cumulativeScore: 12, gamesPlayed: 2, games: [{ gameId: 1, score: 2 }, { gameId: 2, score: 1 }] },
      { id: 3, name: 'Gitte', cumulativeScore: 10, gamesPlayed: 2, games: [{ gameId: 1, score: 1 }, { gameId: 2, score: 3 }] },
      { id: 4, name: 'Anette', cumulativeScore: 9, gamesPlayed: 2, games: [{ gameId: 1, score: 3 }, { gameId: 2, score: 1 }] },
      { id: 5, name: 'Lotte', cumulativeScore: 8, gamesPlayed: 2, games: [{ gameId: 1, score: 2 }, { gameId: 2, score: 2 }] },
      { id: 6, name: 'Peter', cumulativeScore: 7, gamesPlayed: 2, games: [{ gameId: 1, score: 1 }, { gameId: 2, score: 3 }] }
    ],
    games: [
      { gameId: 1, gameDate: '2026-01-01' },
      { gameId: 2, gameDate: '2026-01-08' }
    ]
  })
}));
```

## Task 2: Fix i18n namespace mismatch in SimpleGamesCalendar

**Files:**
- Modify: `src/utils/locales/da.js`
- Modify: `src/utils/locales/en.js`
- Modify: `src/components/SimpleGamesCalendar.jsx` (lines 28, 43, 44)
- Create: `src/test/components/SimpleGamesCalendar.test.jsx`

### Background

`SimpleGamesCalendar` calls `t('gamesList.errorLoading')`, `t('gamesList.noGames')`, and `t('gamesList.startPlaying')` in its error and empty-state renders (lines 28, 43, 44). Those keys belong to the `SimpleGamesList` component's namespace. The `gamesCalendar` namespace exists but lacks these three keys. The fix adds the keys to both locale files and updates the component to reference them.

The locale files are plain JS objects exported as default. Both files have the same structure; keys to add sit inside the `"gamesCalendar"` object.

## Task 3: Fix team avatar rank logic in SimpleTeamStatistics

**Files:**
- Modify: `src/components/SimpleTeamStatistics.jsx:108`
- Create: `src/test/components/SimpleTeamStatistics.test.jsx`

### Background

The spec (documented in `CLAUDE.md` under "Avatar System") requires:

| Team rank | Avatar |
|-----------|--------|
| 1st–3rd   | happy  |
| 4th–9th   | ok (neutral) |
| 10th+     | sad    |

`getRankBasedAvatar(playerName, avatarRank)` in `src/utils/simpleAvatarUtils.js` already implements this mapping, using the numeric `avatarRank` argument:
- `avatarRank === 1` → happy
- `avatarRank === 2` or `3` → ok
- `avatarRank >= 4` → sad

The bug is at `SimpleTeamStatistics.jsx:108`:

```js
const avatarRank = rank <= 3 ? 1 : 4; // Top 3 teams get happy, others get sad
```

This maps ranks 4–9 to `avatarRank=4` (sad) instead of `avatarRank=2` (ok/neutral). Fix: use a three-way ternary.

- [ ] **Step 1: Create the test file with a failing test**

Create `src/test/components/SimpleTeamStatistics.test.jsx`:

```jsx
import { describe, it, expect, vi, beforeEach } from 'vitest';
import { render, screen } from '@testing-library/react';
import { ThemeProvider } from '../../utils/ThemeContext';
import '../../utils/i18n';
import SimpleTeamStatistics from '../../components/SimpleTeamStatistics';

// Intercept which avatarRank the component passes to getRankBasedAvatar
const getRankBasedAvatarMock = vi.fn((playerName) => `/assets/${playerName.toLowerCase()}/ok.png`);

vi.mock('../../utils/simpleAvatarUtils', () => ({
  getRankBasedAvatar: (playerName, rank) => getRankBasedAvatarMock(playerName, rank),
  getPlayerAvatarOptions: () => [],
  getPlayerAvatarPath: () => null,
}));

// Mock SimpleAvatarWithHover to avoid rendering real img elements in tests
vi.mock('../../components/SimpleAvatarWithHover', () => ({
  default: ({ playerName }) => <span data-testid={`avatar-${playerName}`} />,
}));

// Team stats with one team per avatar band: rank 1 (happy), rank 5 (ok), rank 10 (sad)
vi.mock('../../utils/dataUtils', () => ({
  getTeamStatistics: () => [
    { players: ['Jonas', 'Torben'], rank: 1, gamesPlayed: 5, wins: 3, totalPoints: 12, winRate: 60 },
    { players: ['Gitte', 'Anette'], rank: 5, gamesPlayed: 5, wins: 2, totalPoints: 9, winRate: 40 },
    { players: ['Lotte', 'Peter'], rank: 10, gamesPlayed: 5, wins: 0, totalPoints: 5, winRate: 0 },
  ]
}));

const renderWithProviders = (component) =>
  render(<ThemeProvider>{component}</ThemeProvider>);

describe('SimpleTeamStatistics – avatar rank selection', () => {
  beforeEach(() => {
    getRankBasedAvatarMock.mockClear();
  });

  it('passes avatarRank=1 (happy) for a team ranked 1st–3rd', () => {
    renderWithProviders(<SimpleTeamStatistics />);
    const callsForJonas = getRankBasedAvatarMock.mock.calls.filter(([name]) => name === 'Jonas');
    expect(callsForJonas.length).toBeGreaterThan(0);
    expect(callsForJonas[0][1]).toBe(1);
  });

  it('passes avatarRank=2 (ok/neutral) for a team ranked 4th–9th', () => {
    renderWithProviders(<SimpleTeamStatistics />);
    const callsForGitte = getRankBasedAvatarMock.mock.calls.filter(([name]) => name === 'Gitte');
    expect(callsForGitte.length).toBeGreaterThan(0);
    // Before the fix this will be 4 (sad). After the fix it must be 2 (ok).
    expect(callsForGitte[0][1]).toBe(2);
  });

  it('passes avatarRank=4 (sad) for a team ranked 10th+', () => {
    renderWithProviders(<SimpleTeamStatistics />);
    const callsForLotte = getRankBasedAvatarMock.mock.calls.filter(([name]) => name === 'Lotte');
    expect(callsForLotte.length).toBeGreaterThan(0);
    expect(callsForLotte[0][1]).toBe(4);
  });
});
```
